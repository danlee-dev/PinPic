# PinPic 린스타트업 프레임워크

---

## 1. 문제 정의

### 문제/니즈

> "사진을 좋아하는 대학생들은 자신의 사진을 공유하고 인정받을 공간이 부족하고, 다른 사람의 촬영 노하우(장소, 장비, 설정값)를 체계적으로 얻을 수 있는 채널이 없다."

### 문제/니즈의 원인은 무엇일까?

| 원인 1 | 원인 2 | 원인 3 |
|---|---|---|
| 기존 사진 커뮤니티(인스타그램, 에브리타임)는 사진가가 아닌 소비자를 위해 설계됨 | 대학 사진 동아리 안에서만 노하우가 공유되고 외부인은 접근 불가 | 촬영 정보(장소, 카메라, 세팅)를 공유하는 표준화된 포맷이 없음 |

### 근본 원인은 무엇일까?

| 근본 원인 1 | 근본 원인 2 | 근본 원인 3 |
|---|---|---|
| 인스타그램은 사진을 "콘텐츠"로 소비하지 "작품"으로 평가하지 않음. 좋아요는 있지만 기술적 피드백이나 촬영 정보 교환 구조가 없음 | 사진 교육/정보 시장이 "강의" 중심이라 1회성 비용이 높고, 일상적인 촬영 노하우 교환은 존재하지 않음 | 대학생 사진가는 수익화 채널이 전무. 자기 사진으로 돈을 벌 수 있다는 인식 자체가 없음 |

### 문제 정의

> **"어떤 사람의, 어떤 어려움/니즈를 해결하려고 한다."**

**대학생 아마추어 사진가**의, **자신의 사진이 어떻게 찍혔는지 알고 싶어하는 사람들에게 촬영 정보를 체계적으로 전달하고 그 정보에 대한 대가를 받을 수 있는 채널이 없다**는 문제를 해결하려고 한다.

## 2. 페르소나 분석

### 핵심 페르소나 정의

> "어떤 사람의, 어떤 어려움/니즈를 해결하려고 한다."

**핵심 페르소나**: 캠퍼스 사진을 찍는 것을 좋아하는 20대 초중반 대학생. 인스타그램에 사진을 올리고, 다른 사람의 좋은 사진을 보면 "이거 어디서 어떻게 찍었지?"가 즉각적으로 궁금해지는 사람.

### 어려움, 원하는 것, 불만족 사항

- **어려움**: 좋은 사진을 봐도 촬영 장소, 카메라 세팅, 보정 방법을 알 수 없음. DM으로 물어보기엔 부담스럽고, 물어봐도 대부분 무시당함.
- **원하는 것**: "이 사진 어디서 찍었어?", "카메라 뭐야?", "보정 어떻게 했어?"에 대한 즉각적 답변. 그리고 내 사진도 그런 수준으로 인정받고 싶음.
- **기존 솔루션의 불만족**: 인스타는 소비자 위주라 사진가 간 정보 교환이 안 됨. 사진 강의는 비싸고 일회성. 에브리타임은 익명이라 사진 공유 문화가 아님.

### 페르소나 특성

| 정량적 특성 | 정성적 특성 |
|---|---|
| 소득 수준: 대학생 (용돈/아르바이트) | 관심사: 캠퍼스 사진, 필름/디지털 카메라 |
| 지불 가능 금액: 990원~4,900원 (충동구매 임계) | 행동 패턴: 인스타그램 탐색 > 에브리타임 확인 > 카카오톡 공유 |
| 거주지: 서울 (고려대/연세대 근처) | 소비 패턴: 카페 음료 1잔 값 이하면 고민 없이 결제 |
| 직업/학교/전공: 대학생 전 학과 | 콘텐츠 소비: 숏폼(릴스/틱톡) > 인스타 스토리 > 에브리타임 |
| SNS: 인스타그램 (메인), 에브리타임 | 사진가로서의 자의식: "나도 잘 찍는데" 인정받고 싶은 욕구 강함 |

## 3. 가설의 논리성 및 구체성

### 핵심 가설

> **"대학생 사진가들은 자신의 사진과 관련된 특정 정보(촬영 장소, 카메라 세팅, 보정 레시피)에 대해 기꺼이 소액을 지불할 것이다."**

### 가설의 논리 구조

```
전제 1: 대학생은 다른 사람의 좋은 캠퍼스 사진을 보면 "어떻게 찍었는지" 궁금해한다
  ↓ 검증: 투표 단계에서 "촬영 스팟 보기" 티저 버튼 클릭률 측정
  
전제 2: 그 궁금증은 단순한 호기심을 넘어 "돈을 내서라도 알고 싶다"는 수준이다
  ↓ 검증: 990원 페이크 도어 CTR + 이메일 제출률 측정

전제 3: 이벤트 맥락(투표 경쟁, 결과 발표)이 궁금증을 더 강하게 자극한다
  ↓ 검증: 피드 vs 명예의 전당 source별 퍼널 CR 비교
  
결론: 가설이 맞다면, 사진 촬영 정보를 유료 콘텐츠로 패키징한 MVP를 만들 수 있다
```

### 검증 방법: 페이크 도어 (Fake Door)

| 항목 | 내용 |
|---|---|
| **실험 이름** | 제1회 캠퍼스 사진 고연전 — 결과 발표 페이크 도어 |
| **실험 대상** | 이벤트에 투표한 대학생 + 결과 발표를 보러 온 신규 방문자 |
| **실험 기간** | 결과 발표 시각 이후 24~48시간 |
| **페이크 상품** | "TOP 10 비하인드 북" — 상위 10작의 촬영지, 카메라, 렌즈, 세팅값, 보정 레시피, 작가 코멘트 묶음 |
| **가격** | 990원 (대학생 충동구매 임계 이하) |
| **노출 위치** | 명예의 전당 카드 잠긴 메타데이터 / 명예의 전당 상단 CTA bar / photo modal 하단 |
| **측정 지표** | 1차: 페이크 도어 CTR (클릭/노출), 2차: 이메일 제출률 (제출/클릭), 최종: overall CR (제출/모달열기) |
| **성공 기준** | overall CR >= 5% = 가설 강하게 지지 |
| **실패 기준** | overall CR < 2% = 가설 기각 |

### 990원 가격 설정 근거

- 대학생 충동구매 임계치 밑으로 확 낮춰서 **"정보재에 지갑을 여는가"라는 순수한 WTP 시그널**만 뽑기 위함
- 3,900원이면 "비싸서 안 눌렀다"는 노이즈가 섞이지만, 990원이면 그 변명이 사라짐
- 990원에도 안 누르면 → 가설 자체가 틀린 것. 데이터가 깔끔해짐

### 가짜 결제 후 사용자 경험 설계 (윤리)

- "결제 완료" 같은 거짓 문구 절대 사용 X
- 결제 정보(카드번호 등) 안 받음. 이메일 1줄만
- "곧 출시 예정, 사전 신청하면 1순위 발송" 카피로 자연스럽게 연결
- 사전 신청자에게 후속 메일: "가설 검증이었고, 덕분에 정식 개발 착수. 약속대로 무료/할인 발송"

## 4. 실행 및 인사이트

### 실행 과정 (2단계 운영)

#### 4.1 투표 단계 (이벤트 운영)

| 항목 | 내용 |
|---|---|
| **채널** | 에브리타임 양교 게시판 + 인스타그램 바이럴 |
| **참여 방식** | 구글 폼으로 사진 제출 → 관리자 승인 → 웹사이트에 공개 |
| **투표 방식** | Google OAuth 로그인 후 좋아요 (1인 다수 사진 투표 가능) |
| **운영 기간** | 2일 |
| **결과** | 출품작 50장, 총 투표 439표, 참여자 176명, MAU 53 |

**투표 단계에서 이미 얻은 시그널**:

| 측정 | 수치 | 의미 |
|---|---|---|
| 사진 조회 → 스팟 클릭 전환율 | 약 10% | 사진을 본 10명 중 1명은 "어떻게 찍었는지" 궁금해한다 |
| 스팟 클릭 유니크 사용자 | N명 | 이 사람들이 잠재 유료 사용자 |
| 로그인 비율 (조회 기준) | N% | 로그인까지 한 사용자 = 관심도 상위 |

#### 4.2 결과 발표 단계 (가설 검증)

| 항목 | 내용 |
|---|---|
| **결과 페이지** | 명예의 전당 (Hall of Fame) — TOP 10 카드 + 잠긴 메타데이터 + 페이크 도어 |
| **진입 경로 3종** | 피드 사진 클릭 / 명예의 전당 카드 / 상단 CTA bar |
| **퍼널 단계** | 모달 열기 → TOP 10 비밀 클릭 → 990원 사전신청 → 이메일 제출 |
| **측정 방식** | 모든 단계를 source/photo_id/user_id/created_at로 기록 → 어드민 패널에서 실시간 모니터링 |

### 운영 중 만난 문제와 해결

| 문제 | 원인 | 해결 |
|---|---|---|
| Supabase 1000행 limit으로 데이터 누락 (3차례) | REST API 기본 응답 제한 | offset 페이지네이션 + DB view 집계 이전 |
| 원본 이미지 로딩 느림 (모바일) | 2~5MB 원본 직접 로드 | 800px 썸네일 자동 생성 (95% 절감) |
| 단톡방 어뷰징 (시간당 31표) | 조직적 투표 동원 | Peak Velocity Penalty (영구 추적 + 정상 투표로만 회복) |
| 비인기 사진 상단 고착 | Exploration 보너스 과잉 | Quality Gate (6시간 후 평균 30% 미달이면 보너스 감쇠) |
| 표시 순위 조작 증거 노출 | vote_overrides RLS가 열려있음 | admin-only RLS + 서버 API proxy |

### 피드 추천 알고리즘 (자체 개발)

단순 좋아요순/랜덤이 아닌 **"투표를 더 활발하게 하기 위한"** 추천 알고리즘:

```
score = (Wilson * 0.35 + TimeBooth * 0.2 + Exploration * 0.25 + MidBoost * 0.2) 
        * VelocityPenalty * VotedDemote * Jitter
```

- Wilson Score: 통계적으로 공정한 인기도 평가
- Exploration + Quality Gate: 신규 사진 발견 기회 + 비인기 고착 방지
- Peak Velocity Penalty: 어뷰징 영구 추적
- Voted Demote: 이미 투표한 사진 하향 (개인화)
- School Stride Scheduling: 학교 균등 노출

## 6. 인사이트 및 Next Step

### 가설 검증 결과 해석 기준

| overall CR | 해석 | 액션 |
|---|---|---|
| >= 5% | 가설 강하게 지지 | 정식 MVP 개발 착수. "사진 비하인드 북" 상품화 |
| 2~5% | 가설 약하게 지지 | 카피/UI/가격 개선 후 2차 이벤트로 재실험 |
| < 2% | 가설 기각 | 다른 monetization 모델 탐색 (광고/스폰서/오프라인) |

### 가설이 맞다면 — Next Step

1. **MVP 1**: TOP 10 비하인드 북 실제 제작 (사전 신청자에게 무료 발송)
2. **MVP 2**: 사진 업로드 시 촬영 정보 입력 폼 추가 → 유료 열람 구조
3. **MVP 3**: 작가 프로필 + 팔로우 + 유료 촬영 가이드 구독

### 이번 실험에서 배운 것 (실행 인사이트)

- **에브리타임 바이럴 전략이 대학생 타겟에 매우 효과적** → 다음 이벤트도 동일 채널
- **어뷰징은 관심의 증거지만 통제 필요** → 추천 알고리즘 + 어뷰징 패널티로 해결 가능
- **페이지네이션/데이터 limit 문제가 3차례 반복** → Supabase REST API 사용 시 기본 limit 항상 확인
- **이미지 최적화(썸네일)가 모바일 경험의 핵심** → 처음부터 적용해야 함
- **표시 순위 조작은 RLS 잠금까지 해야 완전** → DB 보안은 코드 배포만큼 중요

| 분석 | 자체 측정 테이블 + GA4 + Google Sheets | 어드민 패널에서 실시간 + 스프레드시트에서 오프라인 분석 |
