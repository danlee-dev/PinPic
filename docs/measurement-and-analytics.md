# 측정 체계 및 분석 방법론

핀픽 1회 사진 고연전 운영 중 수집한 사용자 행동 데이터, 그 데이터로 무엇을 검증하려 했는지, 어드민 패널의 어떤 화면이 그 검증을 어떻게 보여주는지 정리한 문서. 발표 자료 베이스로 사용 가능.

---

## 1. 전체 그림

이벤트는 두 단계로 운영된다.

1. **투표 단계 (운영 중)** — 사진 업로드, 피드 노출, 투표
2. **결과 단계 (발표 후)** — 명예의 전당 공개 + 페이크 도어 가설 검증

각 단계에서 별도의 측정 목표와 별도의 데이터 테이블, 별도의 어드민 화면을 갖는다. 어드민 패널의 `관심도` 탭은 이 두 단계를 두 개의 서브탭(`투표`, `결과`) + 종합 분석 서브탭(`분석`)으로 나눠서 보여준다.

## 2. 투표 단계 측정

### 2.1 측정 대상

| 항목 | 어떻게 측정 | 테이블 |
|---|---|---|
| 사진 노출 (impression) | 사진이 피드에서 사용자의 viewport에 들어오는 순간 1회 (세션 내 dedupe) | `photo_views` |
| 스팟 클릭 | 사용자가 photo modal 안에서 "촬영 스팟 및 설정값 보기" 티저 버튼을 누른 시점 | `photo_clicks` |
| 투표 / 투표 취소 | 사용자가 사진에 좋아요를 누른 시점 | `votes` |

각 row에는 `photo_id`, `user_id` (로그인 시), `created_at`을 기록한다. `user_id`가 NULL인 row는 비로그인 익명 사용자.

### 2.2 가설 (투표 단계)

- **H1.** "사진을 본 사람"과 "촬영 정보가 궁금해서 클릭한 사람"의 비율은 사진별로 크게 달라질 것이다 → 어떤 사진이 단순 미적 호기심을 넘어서 "어떻게 찍었는지 알고 싶다"는 깊은 호기심을 자극하는지 식별 가능
- **H2.** 학교/동아리/개인별로 노출 대비 클릭 전환율이 달라질 것이다 → 누가 사진가로서 가장 주목받는지 분석

### 2.3 어드민 화면 (`관심도` → `투표`)

#### 전체 통계 카드
- **총 조회수 / 총 스팟 클릭수 / 전환율** — 이벤트 전체 모멘텀의 압축 지표
- **사진 열람 N명 / 스팟 클릭 N명 / 미클릭 N명** — 동일한 카운트지만 unique user 단위로 제공해서 "한 명이 여러 번 본 것"과 "여러 명이 본 것"을 구분
- **로그인 vs 비로그인 (조회/클릭)** — 로그인 비율이 곧 회원 전환 모멘텀

#### 사진별 / 사용자별 리스트
- **사진별**: 각 출품작의 조회수, 클릭수, 전환율, unique viewer/clicker. `조회순` 또는 `전환율순` 정렬 가능. 닉네임/동아리 검색.
- **사용자별**: 가장 활발히 활동한 유저의 이메일/조회/클릭/전환율. 가장 충성도 높은 코어 유저 식별용.

이 화면에서 얻을 수 있는 인사이트:

- "조회는 많은데 클릭이 거의 0인 사진" → 시각적 매력은 있지만 정보 욕구는 안 자극
- "조회는 적은데 클릭률 50% 넘는 사진" → 코어 팬을 가진 사진. 고품질 시그널
- "특정 유저가 100건 넘게 사진을 열어봄" → 슈퍼유저 후보, 차후 알파 테스터 풀

### 2.4 데이터 흐름

```
브라우저 (recordPhotoView/recordPhotoClick)
  ↓ insert
Supabase: photo_views, photo_clicks
  ↓ DB view 집계
engagement_totals, engagement_by_photo, engagement_by_user (RPC)
  ↓ fetchEngagementStats()
어드민 패널 (관심도 → 투표)
  ↓ 동시 미러링
Apps Script analytics-sync.js → Google Sheets
```

집계는 가능한 한 DB view에서 수행 (Supabase REST API의 1000행 limit 회피). 이 결정의 이유는 [issues-and-solutions.md](issues-and-solutions.md)에서 별도 설명.

## 3. 결과 단계 측정 (페이크 도어)

### 3.1 검증하려는 핵심 가설

> **출품자(특히 자기 사진에 대한 추가 정보를 원하는 사람)는 정보재에 돈을 지불할 의향이 있다. 단, 990원 임계선에서 의미 있는 클릭 전환이 발생할 것이다.**

이 가설을 검증하기 위해 결과 발표 페이지에 **페이크 도어 (Fake Door) 패턴**을 도입했다. 실제 결제 시스템 없이 "TOP 10 비밀 (장소·세팅값) 전부 열기 · ₩990" 버튼을 노출하고, 클릭한 사용자에게 사전 신청 모달을 띄워 이메일을 수집한다. 클릭률과 이메일 제출률이 곧 실측 WTP (Willingness To Pay) 시그널.

### 3.2 측정 지점 (5단계 funnel)

각 단계마다 어디서, 어떤 사용자가, 언제, 어떤 사진과 함께 일어났는지 기록한다.

| 단계 | 트리거 | 저장 테이블 | source 값 |
|---|---|---|---|
| 1. 사진 모달 열림 | 사용자가 사진 카드를 탭함 | `photo_modal_opens` | `feed` 또는 `hall_of_fame` |
| 2-A. 카드 unlock 클릭 | 명예의 전당 카드 안 unlock 텍스트 | `fake_door_clicks` | `inline_card_1`..`inline_card_10` |
| 2-B. photo modal unlock | photo modal 하단 골드 버튼 | `fake_door_clicks` | `photo_modal` |
| 2-C. 상단 CTA 클릭 | 명예의 전당 헤더 위 큰 CTA 바 | `fake_door_clicks` | `inline_bar` |
| 3. 사전신청 클릭 | 페이크 도어 모달의 "990원에 사전신청 하기" | `fake_door_clicks` | `${parent_source}__pitch_continue` |
| 4. 이메일 제출 완료 | 모달의 "사전 신청 완료" | `fake_door_clicks` + `waitlist` | `${parent_source}__email_submit` |

각 row는 `user_id` (로그인 시) / `photo_id` (해당 시) / `source` / `created_at`. `__` 구분자로 어느 진입 경로의 어느 단계인지 한 row에서 식별 가능 → 단일 테이블에서 다양한 funnel을 재구성할 수 있다.

### 3.3 비로그인 사용자도 동일하게 측정

- 로그인 강제 X. 페이크 도어 모달에서 이메일 1줄만 받음
- `user_id`가 NULL인 row도 모두 카운트
- 단, 비로그인 사용자는 voter cohort 비교에서 제외 (voter_id가 세션 fingerprint라 auth.users uuid와 매핑 불가)

### 3.4 어드민 화면 (`관심도` → `결과`)

#### 전체 카드
- **총 클릭** — 페이크 도어 진입(모든 source 합계)
- **로그인 유저** — unique 로그인 유저 수
- **비로그인** — 익명 클릭 row 수

#### 소스별 클릭
- 어디서 페이크 도어로 들어왔는지 source별 row count
- `inline_bar` 우세 → 결과 화면 상단 CTA가 가장 강력한 진입점
- `inline_card_*` 우세 → 카드 안 텍스트가 더 자연스럽게 클릭 유도
- `photo_modal` 우세 → 호기심으로 사진을 열어본 유저가 결제 의향 더 높음

#### 사진별 / 사용자별 클릭
- 어떤 사진과 어떤 사용자가 가장 많이 결제 의향 시그널을 보냈는지 분석

### 3.5 가설 검증 임계값

| CTR | 해석 | 액션 |
|---|---|---|
| ≥ 5% | 가설 강하게 지지 | 정식 MVP 개발 착수 |
| 2~5% | 가설 약하게 지지 | 카피·UI 개선 후 재실험 |
| < 2% | 가설 기각 | 다른 monetization 모델 탐색 |

### 3.6 결제 의향 지표가 가지는 한계

- "클릭"은 호기심일 수도 있고 실제 결제 의향일 수도 있다 → 진짜 시그널은 **이메일 제출 단계**
- 여러 단계를 통과하는 사용자 = 깊은 의향. CR(이메일 제출 / 모달 열림)이 진짜 핵심
- 990원 임계는 충동 구매 한도 아래 → "비싸서 안 눌렀다"는 변명 차단 → 가설 자체에 대한 깨끗한 시그널

## 4. 종합 분석 (`관심도` → `분석`)

투표 단계와 결과 단계를 **연결**해서 보는 화면. 단순히 두 데이터를 나란히 두는 게 아니라, 같은 사용자가 두 단계에서 어떻게 행동했는지를 묶어 보여줘서 더 강한 인과 신호를 뽑는다.

### 4.1 핵심 헤드라인

화면 최상단 카드 — 발표용 한 줄 요약:

- **모달 → 사전신청 전환율 (overall CR)**: 모든 모달 열림 대비 모든 사전신청 제출의 비율
- **사전신청 누적 카운트**: 절대수치로 본 WTP 시그널 양

### 4.2 소스별 퍼널

세 가지 진입 경로를 각각 별도 funnel로 시각화:

**A. 피드 진입 → 사전신청 퍼널**
```
사진 모달 열기 (피드)
  ↓ N%
TOP 10 비밀 클릭
  ↓ N%
990원 사전신청 클릭
  ↓ N%
이메일 제출 완료
```

**B. 명예의 전당 진입 → 사전신청 퍼널** (동일 구조, 다른 source)

**C. 명예의 전당 상단 CTA 직접 진입 퍼널**
```
Hall of Fame 진입
  ↓
상단 CTA 클릭
  ↓
990원 사전신청 클릭
  ↓
이메일 제출 완료
```

각 단계 옆 % 컬러 코딩: ≥30% green / 15~30% yellow / <15% red. 즉 어느 단계에서 가장 큰 drop-off가 발생하는지 한눈에 보인다.

**얻는 인사이트**:
- A vs B 비교 → 결과 컨텍스트(명예의 전당)가 WTP를 자극하는지 시사
- B vs C 비교 → 사진을 보고 들어온 사용자 vs 곧장 CTA 누른 사용자 중 누가 더 진성 의향
- 어느 단계에서 drop-off가 큰지 → 카피/UI 개선 우선순위

### 4.3 코호트 비교: 투표자 vs 비투표자

- 로그인 유저 universe 안에서 한 번이라도 투표한 사용자 (cohort A)와 안 한 사용자 (cohort B)로 나눔
- 각 코호트의 페이크 도어 클릭률 + 이메일 제출률 비교

| 지표 | 해석 |
|---|---|
| A의 클릭률 > B의 클릭률 | 참여도(투표)와 WTP의 양의 상관 → 코어 팬에게 유료 기능을 우선 풀어야 한다 |
| A ≈ B | 투표 여부와 무관한 일반적 호기심 → 페이크 도어 카피가 모든 사용자에게 통한다 |
| A < B | 의외의 신호. 익명 호기심 그룹이 진짜 결제 의향을 갖는다 = 회원 가입 유도가 오히려 결제 마찰일 수 있음 |

### 4.4 사진별 사전신청 자극도

- 사진별로 (모달 열림 횟수, 페이크 도어 클릭 횟수, 비율)
- 비율이 가장 높은 사진 = 단순 시각적 매력을 넘어 "촬영 노하우가 궁금한" 사진
- 이 사진의 작가는 잠재 콘텐츠 크리에이터 후보. 다음 라운드 운영 자산.

### 4.5 시간대별 활동 분포

- 1시간 단위 bucket으로 modal opens / fake door clicks / email submits
- 결과 발표 시각 직후의 피크가 얼마나 큰지, 얼마나 빨리 빠지는지 측정
- 가설: 발표 직후 1~3시간이 절대 피크 → 다음 이벤트 운영 시 이 윈도우에 알림 푸시 집중

### 4.6 이메일 도메인 분석

- 사전신청 이메일을 도메인별로 (gmail, naver, 학교 도메인 등) 카운트
- 인구통계 추정 + 마케팅 채널 우선순위 (학교 도메인 비율이 높으면 학교 커뮤니티 광고가 효과적)

## 5. 데이터 저장 + 외부 미러링

### 5.1 Supabase 테이블 (이번 이벤트 신규)

| 테이블 | 용도 |
|---|---|
| `photo_modal_opens` | 사진 모달 열림 (피드/명예의 전당 source 구분) |
| `fake_door_clicks` | 페이크 도어 모든 단계 (CTA 클릭 / pitch_continue / email_submit) |
| `waitlist` | 사전 신청 이메일 + product_id + source |

### 5.2 RLS 정책

- **insert**: 누구나 (비로그인 포함)
- **select**: 어드민만 (`admins` 테이블 가입자)

이로 인해:
- 데이터 소유권은 어드민
- 일반 유저는 측정 점수를 조작하거나 다른 사람의 사전신청 내역을 볼 수 없음
- 비로그인이 끝까지 funnel 통과 가능

### 5.3 Google Sheets 미러링

`scripts/analytics-sync.js`가 Apps Script 시간 트리거로 주기 동기화:

| Sheet | 내용 |
|---|---|
| 전체 통계 | 총 조회/클릭/전환율 |
| 사진별 | 출품작별 조회·클릭·전환율 |
| 사용자별 | 유저별 조회·클릭·전환율 |
| 조회 로그 | photo_views raw |
| 클릭 로그 | photo_clicks raw |
| 페이크도어 전체 | 페이크 도어 총량/로그인/비로그인/사진 |
| 페이크도어 소스별 | source별 클릭 카운트 |
| 페이크도어 사진별 | 사진별 페이크 도어 클릭 |
| 페이크도어 사용자별 | 유저별 페이크 도어 클릭 |
| 페이크도어 로그 | fake_door_clicks raw |
| 모달 열기 로그 | photo_modal_opens raw |
| 모달열기 소스별 | source별 모달 열림 카운트 |

`fetchTable`은 Supabase REST API의 1000행 limit을 회피하기 위해 offset 페이지네이션을 적용. 이 결정 배경은 [issues-and-solutions.md](issues-and-solutions.md) 참조.

## 6. 어드민 데이터에서 무엇을 도출할 수 있나 — 예시 시나리오

### 시나리오 A. 가설 강하게 지지

```
overall CR (이메일/모달열림) = 8.4%
inline_bar funnel: 100 → 67 → 41 → 28
hall_of_fame funnel: 312 → 89 → 51 → 32
voter cohort 클릭률: 22%
non-voter 클릭률: 9%
top photo CR: 17%
```

→ "정보재 페이드 의향은 존재한다. 특히 투표자 코호트에서 두 배 이상 강하게. 명예의 전당이 아닌 inline CTA 진입의 컨버전이 가장 깔끔하므로 다음 MVP는 이 진입 패턴을 첫 화면으로 가져간다."

### 시나리오 B. 가설 약하게 지지

```
overall CR = 2.1%
funnel drop-off가 모달 → unlock 단계에 집중
이메일 제출 단계 CR은 정상 (35%)
```

→ "WTP는 존재하지만 unlock 진입까지 가는 사용자가 적다. 카피와 위치를 개선해서 모달 → unlock 단계의 visibility를 올리고 재실험."

### 시나리오 C. 가설 기각

```
overall CR < 1%
모든 funnel에서 unlock 클릭 자체가 거의 없음
```

→ "990원이라는 극단적으로 낮은 가격에서도 클릭이 안 나온다. '정보재에 대한 결제 의향' 가설 자체를 폐기. 다음 monetization은 광고/스폰서십/오프라인 굿즈 등 다른 모델 탐색."

## 7. 윤리

페이크 도어는 사용자 신뢰에 영향을 줄 수 있다. 이번 운영의 안전장치:

1. **거짓말하지 않음** — "결제 완료" 같은 문구는 절대 사용하지 않고, "곧 출시돼요 / 사전 신청" 카피만 사용
2. **이메일은 약속의 매개체** — 사전 신청자에게는 실제로 출시 알림 + 무료/할인 발송 약속을 지킴
3. **결제 시도 시 카드 정보를 받지 않음** — 어떤 결제 정보도 입력 단계가 없음, 이메일 한 줄만
4. **검증 후 솔직 reveal** — 사전 신청자에게 후속 메일에서 "사실 가설 검증이었고, 덕분에 정식 개발에 들어갑니다"라고 솔직하게 공개

## 8. 이번 측정 체계의 한계

- **익명 voter_id를 auth uid로 매핑 불가** → cohort 분석은 로그인 유저로만 가능
- **사진 수가 작음 (50장)** → 사진별 funnel은 통계적 신뢰도가 낮음, 정성적 신호로만 해석
- **운영 기간이 짧음 (이틀)** → 시간대별 분포 패턴이 단일 사이클만 관찰됨, 장기 패턴 추출 불가
- **결과 발표 직후가 모든 측정의 피크** → 가설 검증 자체는 발표 후 24~48시간 내 데이터로만 결정 가능

이 한계들은 다음 라운드에서 운영 기간 확장 + 사용자 풀 확장 + 확률적 매핑(IP/디바이스 fingerprint)으로 점진 개선 예정.
